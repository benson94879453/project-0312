# 2026-03-16 開發日誌

## 今日完成
- 調整座位架構：座位狀態改由 `Chair` 持有，不再由 `Table` 存放 seat status。
- `Chair` 新增狀態與 API：
  - `seat_state`（`available` / `occupied`）
  - `occupied_by`
  - `reserve(actor)` / `release(actor)` / `is_available()` / `is_occupied_by(actor)`
- `Table.reserve_seat/release_seat` 改為協調層：透過 `connected_chairs` 呼叫 chair API。
- `Table` 目前只保留座標與繪製資訊（debug seat），不保管 seat 狀態。
- `Table.interact(actor)` 維持可測流程：互動可佔位/釋放，無空位回傳 `no_free_seat`。

## 影響檔案
- `game/scripts/actors/chair.gd`
- `game/scripts/actors/table.gd`

## 現在可直接測試
- 玩家靠近桌子按 `E`：第一次互動佔位，再按一次釋放。
- Console 觀察：`result=reserved/released/no_free_seat`。
- Debug seat 顏色會跟著對應 chair 狀態切換。
