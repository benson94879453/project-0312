# 2026-03-16 開發日誌

## 本輪目標（依你提供三階段）
- 階段一：建立 Customer 狀態流轉骨架。
- 階段二：讓 Table 成為資料中心（customers/orders/foods）。
- 階段三：讓 Player 可把手上食物送到桌上並觸發顧客吃飯。

## 本輪完成
- `Table` 轉為資料中心：
  - `current_customers`
  - `expected_orders`
  - `foods_on_table`
- `Table` 新增 API/訊號：
  - `register_order(order)`
  - `receive_food(food_item)`
  - `order_registered(customer, food_id)`
  - `food_received(food)`
  - `order_served(customer, food)`
- `Table.interact(actor)` 新增送餐邏輯：
  - 若互動者有 `get_held_food/try_consume_held_food`，就嘗試 `receive_food`
  - 成功才消耗玩家手上餐點
- 新增 `OrderData`（資料結構）
- 新增 `Customer` 狀態機骨架：
  - `SEEKING_SEAT -> ORDERING -> WAITING_FOOD -> EATING -> LEAVING`
  - 可 `start_lifecycle(table)` 自動訂位、點餐、等待上菜、吃完離席
  - 透過 `Table.order_served` 進入 `EATING`
- `Player` 新增手持餐點欄位與方法：
  - `held_food`
  - `get_held_food()` / `try_consume_held_food()` / `set_held_food()`
  - `debug_start_food_id` 預設先拿 `coffee`
- 新增測試場景節點：
  - `game/playground/customer.tscn`
  - `test.tscn` 已放入 `Customer`，可 `auto_start_on_ready` 指向測試桌子

## 目前可驗證流程
1. 進入 `test.tscn`。
2. Customer 會自動進入 `WAITING_FOOD`（成功訂位＋下單）。
3. 玩家靠近桌子按 `E`，若手上是 coffee 且命中訂單：
   - 餐點進入 `foods_on_table`
   - Customer 轉入 `EATING`
   - 計時後進入 `LEAVING` 並釋放座位
